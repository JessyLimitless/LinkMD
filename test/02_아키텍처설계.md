# 시스템 아키텍처

## 전체 구조

```
┌─────────────────────────────────┐
│          CLIENT (Browser)        │
├─────────────────────────────────┤
│         API GATEWAY (Express)    │
├─────────────────────────────────┤
│        CORE ENGINE LAYER         │
├─────────────────────────────────┤
│         STORAGE LAYER            │
└─────────────────────────────────┘
```

## 핵심 설계 원칙

Pandoc-First 전략을 채택하여 변환 엔진은 Pandoc에 위임한다.

## API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| POST | /api/upload | 파일 업로드 |
| POST | /api/convert | 변환 실행 |
| GET | /api/download/:id | 다운로드 |

## Pandoc 명령어

```bash
pandoc input.md -o output.docx --reference-doc=template.docx
```
